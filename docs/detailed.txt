Here are the steps taken by Cowboy to generate a unit test:
1. Create repo
2. Find all TestModules inside repo. A TestModule is just a container for a set of tests. In Python/Pytest, it is either the source file of the test itself or its a class that implements its testcases as methods. Each TestModule will be associated with a single test file.

test_file.py

import pytest
def test1(): 
  ...
  
def test2():
  ...

class TestClass:
  def test_meth1():
    ...
  def test_meth2():
    ...

So for the above, two TestModules will be created, along with their associated tests 
TM1: test_file.py --(tests)--> test_file.py::test1, test_file.py::test2
TM2: TestClass    --(tests)--> TestClass::test_meth1, TestClass::test_meth2 

3. For each TestModule, prompt a LLM with the following to generate a batch of tests
"""
Given the following existing test suite:
{existing_tests}
For the source file(s):
{source_file for source_file in from_step3}
Extend it with additional unittests that improve coverage
"""

4. Get the coverage contributed by each new testfile. Throw away all those that error'd out or failed to contribute coverage

5. Write the coverage improving tests to the test file, loop 4-6 until some iterations N has been reached. This way of incrementally adding new generated tests to the prompt context pushes the LLM to consider new uncovered test conditions

6. Collect generated tests, each contributing new coverage. Win!
